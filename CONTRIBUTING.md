# Contributing to Zenify

First off, thank you for considering contributing to Zenify! It's people like you that make Zenify such a great state management and architecture tool for Flutter.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](https://github.com/sdegenaar/zenify/issues) first. If you don't see anything similar, feel free to open a new one.

## Development Setup

1. Fork the repository and clone it locally.
2. Run `flutter pub get` to install dependencies.
3. Run the test suite using `flutter test` to ensure everything is working correctly before you start.

## Pull Request Process

1. Ensure any new feature or fix includes corresponding tests. Zenify maintains a strict 95%+ coverage rule.
2. Update the `README.md` or `doc/` files with details of changes to the interface or behavior.
3. Ensure the CI pipeline passes (Lints, formatting, and tests).
4. Provide a clear description in your Pull Request using the provided PR template.

## Code of Conduct

Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.
